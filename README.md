MySQL Workbench Project - E-commerce Database

This project demonstrates the management of a sample e-commerce database using MySQL Workbench. The database contains tables for customers, products, orders, and order items.

Database Structure

Customers: Stores customer information such as name, email, and address.

Products: Contains product details including name, price, and description.

Orders: Records orders placed by customers, including total amount and order date.

Order Items: Maintains details of products in each order, including quantity.

Features Implemented

Creation of tables for customers, products, orders, and order items.

Insertion of sample data for customers, products, and orders.

Retrieval of customer orders with details.

Retrieval of orders exceeding a specific total amount.

Calculation of average order total using normalized database tables.

Database normalization to separate order items from orders.

How to Use

Open MySQL Workbench and connect to your MySQL server.

Create a new schema or use an existing one.

Create the tables (customers, products, orders, order_items).

Insert sample data into the tables.

Use queries to retrieve information about customers, orders, and products.

Project Purpose

This project demonstrates how to design a relational database, normalize tables, and run queries to manage and analyze data in an e-commerce scenario.
